# 2026-05-07

## Summary

- Completed the primary weapon and passive ruleset migration that had been planned as the next large compiler/runtime cleanup.
- Added explicit primary weapon metadata in `data/weapons.json`:
  - `tags`
  - `primary_behavior`
  - standard primary stats inside `base_stats`
  - migrated `levels` data with `effects` and `overrides`
- Reworked `RunState` so primary compilation now uses the new standard-stat path while secondaries intentionally remain on the legacy path.
- Split passive compilation into separate domains:
  - global effects
  - primary per-slot effects
  - secondary per-slot effects
- Added optional passive `requires_tags` filtering so projectile-tagged effects only apply to compatible primaries.
- Migrated primary-affecting passive content onto `effects` while keeping secondary legacy behavior isolated behind `passive_effects`.
- Replaced the old player projectile fire payload with a config-dictionary path for primary fire without changing the enemy fire path.
- Added compiled projectile `range` and `area` support to the player-primary projectile path.
- Added projectile `pierce` support and a projectile-tagged pierce passive.
- Made aim assist use compiled weapon range instead of the old fixed snap radius.
- Removed the legacy primary schema/runtime path after the new primary compiler was in place.
