# 2026-05-11

## Summary

- Implemented an enemy tempo, pathing, and combat-juice follow-up pass on top of Patch 13 validation work.
- Replaced the old delayed blocked-time obstacle detour with immediate feeler-based steering in enemy movement.
- Added capped local enemy separation so Swarm-heavy rooms spread instead of bunching into obstacle corners.
- Replaced the old six-marker wave spawn system with arena-wide valid spawn sampling.
- Sampled wave spawns now respect obstacle geometry, arena bounds, spawn-to-spawn spacing, and a minimum distance from players.
- Added `capture_the_hill` as a live combat objective path and shifted one recipe to use it instead of generators.
- Hill rooms now use a sampled control zone plus normal combat-wave pressure instead of generator-owned objective spawning.
- Restructured the old debug single-room path toward an `Encounter Builder` flow:
  - home `Debug` now opens a builder-oriented setup
  - single-room launches auto-enter the configured encounter
  - room completion/failure now returns directly to the builder instead of a one-node debug map
- Raised baseline move speeds across the full enemy roster:
  - `Chaser`: `108 -> 145`
  - `Spitter`: `84 -> 110`
  - `Charger`: `88 -> 125`
  - `Bruiser`: `68 -> 92`
  - `Boss`: `82 -> 105`
- Tightened melee attack timings:
  - `Chaser` lunge windup `0.16 -> 0.10`, cooldown `1.25 -> 1.0`
  - `Charger` windup `0.34 -> 0.20`, cooldown `2.3 -> 1.8`
  - `Bruiser` windup `0.45 -> 0.30`, recovery `1.8 -> 1.4`, cooldown `2.5 -> 1.9`
- Strengthened enemy attack/kill feedback:
  - heavier particle scaling for impact, death, and explosion bursts
  - short attack trails during active lunge / charge / slam windows
  - stronger enemy-hit, enemy-death, and explosion hitstop / trauma
  - `ScreenShake` raised to `34` max with faster decay `3.4`
- Changed the default profile screen-effects setting from `Off` to `Full` so the current combat-feedback pass is visible without manual setup.
- Followed up with an encounter-builder/runtime review pass:
  - fixed single-room encounter clear flow so builder launches return to the builder instead of the full run-summary path
  - filtered builder objective selection down to the current non-generator focus
  - throttled repeated in-room HUD/debug refresh work
  - replaced per-enemy scene-tree group scans in steering/separation with direct combat-owner target lists
  - cleared the remaining GDScript shadowing warnings in runtime/UI helper code

## Outcome

- Enemy movement should now react before wall contact instead of stalling and then jerking sideways.
- Obstacle-heavy Swarm rooms should now read as active pressure instead of compressed enemy clumps.
- Faster enemy pace is now paired with clearer hit/kill readability instead of only increased threat.
- Wave enemies are no longer forced to enter from six predictable marker slots, so blocked layouts should not repeatedly jam the same corners.
- Generator rooms are no longer the main objective focus for near-term combat validation; hill-control rooms now cover that iteration space better.
- Encounter Builder now behaves more like a true iteration loop instead of accidentally falling back into run-summary behavior.
- Headless parse is now clean again after the warning sweep, so editor/script reload noise is reduced.
